# Dispatch Plan: Fix Run (2026-04-29)

## North Star
Best agent-first proxy MCP on the internet. Real-world tested, production-ready.

## What We're Fixing
From RCA after 31 real-world tests across 3 proxy types:

1. **BUG 2:** 404→PAGE_NOT_FOUND misclassifies anti-bot responses (Amazon returns 404 as bot-block)
2. **TLS instruction:** Bad domains return TLS_ERROR via proxy — agent_instruction should mention DNS possibility
3. **Crawl credits:** Double-fetch when include_content=true + format=markdown costs 2x but reports 1x
4. **links_found:** Field reports "new unseen links" not "total links on page" — misleading
5. **Extract gaps:** BBC image=null, links=[] — fallback extraction needs improvement

## Agents: 5 Fixers → 5 Reviewers → 2 Verifiers

### Fixer 1 — errors.ts (BUG 2 + TLS instruction)
**File:** src/errors.ts
**Fix A:** Remove the 404→PAGE_NOT_FOUND special case. Amazon/many sites use 404 as bot-block through proxies. Revert to: let 404 fall through to the generic 4xx→BOT_DETECTION_SUSPECTED handler. Remove PAGE_NOT_FOUND from types.ts. Update proxy://error-codes resource in index.ts.
**Fix B:** Add to TLS_ERROR agent_instruction: "If this is an unrecognized domain, verify the domain exists — proxy-side DNS failures may surface as TLS errors."

### Fixer 2 — crawl.ts (credits + links_found)
**File:** src/tools/crawl.ts
**Fix A:** When include_content=true AND format=markdown, the code fetches markdown (for content) then raw (for links). The raw fetch may NOT be cached (different cache key: url|raw|country vs url|markdown|country). Track actual fetch count and report accurate credits_estimated.
**Fix B:** Rename `links_found` to `new_links` in CrawlPageResult. Add `total_links` field that counts ALL <a href> tags on the page before deduplication.

### Fixer 3 — extract.ts (image/links gaps)
**File:** src/tools/extract.ts
**Fix A:** Image extraction: add fallback to first <img> tag with src if og:image and JSON-LD image both fail.
**Fix B:** Links extraction: extractAllLinks currently only returns absolute http URLs. Many sites use relative URLs. Resolve relative URLs against the page's base URL (pass URL to extractField and use new URL(href, baseUrl)).
Note: extractField signature may need to change to accept `url` parameter for relative URL resolution.

### Fixer 4 — README.md (test results + limitations)
**File:** README.md
**Task:** Add a "Real-World Test Results" section and "Known Limitations" section.
- Test results: 22/26 pass across 3 proxy types
- Known limitations: (1) Datacenter IPs blocked on anti-bot sites, (2) Proxy DNS → TLS errors on bad domains, (3) Residential sticky sessions may require account-specific endpoint config
